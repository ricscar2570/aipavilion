# Phase 4 report: controlled-pilot operations foundation

## Objective

Turn the multi-tenant Phase 3 codebase into a retained staging and pilot foundation with infrastructure-managed frontend delivery, perimeter controls, SaaS billing, email invitations, migrations, backup, observability, privacy operations and accessible frontend foundations.

## Implemented

### Persistent staging architecture

- Generated retained backend template restricted to `staging` and `production`.
- DynamoDB point-in-time recovery and retain policies in the pilot template.
- Cognito MFA optional and deletion protection enabled.
- Versioned schema-migration table, plan/apply/verify tooling and Phase 4 migration.
- Private encrypted S3 frontend origin with CloudFront Origin Access Control.
- CloudFront TLS, SPA fallback, cache policy, access logging and response security headers.
- Exact API origin in Content Security Policy.

### Security and abuse controls

- AWS WAF managed common, known-bad-input and IP-reputation rule groups.
- Per-IP anonymous rate rule.
- Cloudflare Turnstile integration for anonymous lead submission, with isolated simulated development mode.
- Local Tailwind build; optional Stripe and Turnstile scripts load only when required.
- Weekly security workflow with dependency and source-secret gates.
- Access-token authorization preserved for protected APIs.

### Billing and entitlement synchronization

- Organization billing summary endpoint.
- Stripe subscription Checkout and Billing Portal sessions.
- Separate signed billing webhook.
- Conditional webhook event claiming and replay suppression.
- Entitlement plan, state, limits and features synchronized from subscription events.
- Simulated billing retained only for development testing.

### Invitation delivery

- SESv2 invitation delivery with configuration-set tags.
- Invitation list, resend and revoke operations.
- Delivery, bounce, complaint and reject event processing through SNS.
- Delivery state and audit history stored against the invitation.
- SPA and email invitation links use the same route.

### Product and privacy operations

- Organization onboarding profile fields.
- Member grant, suspension, reactivation and removal.
- Event duplication and archival, including public stand withdrawal.
- Organization audit viewer.
- Authenticated personal data export with payment internals removed.
- Audit retention configuration and engineering data inventory.

### Reliability and operations

- CloudWatch API error and p95 latency alarms.
- WAF block-volume alarm and operations dashboard.
- SNS alert channel.
- Daily AWS Backup plan with retained vault.
- DynamoDB point-in-time restore drill script.
- Monthly actual/forecast budget alerts.
- Hourly staging synthetic workflow.
- Protected staging deployment workflow using GitHub OIDC.
- Incident response and pilot service-objective documents.

### Accessibility foundation

- Tailwind compiled locally rather than injected from a CDN.
- Skip link and reliable body-ready state.
- Focus and reduced-motion stylesheet rules.
- Live status regions in role portals.
- Manual WCAG 2.2 AA checklist and assistive-technology matrix.

- 61 Node.js files passing syntax checks;

## Verification boundary

This environment does not provide AWS CLI, SAM CLI or deployment credentials. Therefore the report does not claim that CloudFormation, WAF association, SES delivery, Stripe test webhooks, AWS Backup restore, Cognito, API Gateway or the deployed Playwright journey were executed here.

The repository contains the automated deployment, migration, synthetic, restore and browser journeys required to obtain that evidence in a controlled AWS staging account.

- SES production-access approval and real recipient tests;
- measured RPO/RTO restoration evidence;
